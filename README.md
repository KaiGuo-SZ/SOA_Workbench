# 策略运营分析工作台 V2（体验优化版）

本版本只优化产品架构、界面和分析路径，不调整任何指标口径、聚合逻辑或异常规则。

## 本次调整

- 将原来的七个并列页签重组为“经营驾驶舱 / 组织分析 / 数据管理”三级导航。
- 保留项目、营期、大组、小组、班长和数据健康全部分析维度。
- 增加页面当前位置、分析目的和统一的“结果 → 对比 → 过程 → 明细”路径提示。
- 将全局筛选分为基础筛选与组织筛选，减少首屏操作负担。
- 常驻展示当前生效的数据范围，降低用户误读筛选口径的风险。
- 在总览异常与行动建议中增加直达对应分析页的入口。
- 桌面端使用左侧工作台导航，中小屏自动切换为横向导航。
- 营期分析归入经营驾驶舱，兼容单项目负责人的日常经营视角。
- 大组与小组的组内趋势改为按开营时间和营期顺序连接，不再按周聚合。
- 营期趋势总览强制展示全部营期横坐标标签。
- 所有以营期为横坐标的趋势图统一锁定完整类目，不再由图表组件自动跳过标签。
- 营期热力图调整为两排、每排三张，并新增 D3-D7 到播转率和个销转率。
- 大组、小组“组内趋势”支持在营期维度与周维度之间切换，图表与趋势明细同步更新。
- 图表改为按当前页签按需渲染，减少首次进入和筛选后的绘图负担。
- Plotly 与 PapaParse 改为项目内本地依赖，降低外部网络造成的加载失败和转圈风险。
- 统一营期横轴标签密度、数字等宽显示、卡片层级和热力图好坏色阶。
- 筛选存在未应用变更时提供明确视觉反馈，无变更时禁用重复应用。
- 经营总览重构为“进行中营期昨日干预 + 最近结营3期复盘”：过程异常固定检查当前Day减1，ROI目标0.8、人均产能目标6万元，其余关注指标仅做期间变化比较。
- 昨日异常支持项目、大组、小组、班长四级：前三层使用历史最近5期同Day基准，班长使用同营期同Day均值；预警按偏离5%/10%/20%分为黄/橙/红，回复率与回复时长优先执行分Day固定预警线。
- “当前运营营期”新增一键复制昨日播报：自动汇总风险等级、关键指标、主要阻塞、重点异常和今日动作，并按指标所处运营阶段处理“暂未进入”。

## 使用方式

直接打开 `index.html`，按原流程上传班级明细 CSV 后开始分析。

原版本仍保留在项目根目录，可用于功能与体验对照。
